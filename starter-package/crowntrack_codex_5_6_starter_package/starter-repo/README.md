# CrownTrack ADV

CrownTrack is an offline-first Android/iOS adventure-bike application in staged development. Stage 2.2 established native local persistence. Stage 3A adds explicit device identity and authenticated offline pairing. Pairing authenticates identity and invitation integrity; it does not encrypt location content.

Not yet implemented: signed location messages, verified ride-group membership, end-to-end encryption, production accounts/cloud authentication, real GPS/background tracking, networking, Bluetooth, Nearby Connections, Meshtastic, MapLibre, or production map data.

## Setup

```powershell
npm install
npm run typecheck
npm run lint
npm test
npm run test:relay
npx expo-doctor
npx expo install --check
```

## Run the mobile shell

```powershell
npm run start
```

The tabs are Map, Packs, Plan, Crew, and SOS. Native Android/iOS builds use Expo SQLite; web intentionally uses the in-memory repository.

## Stage 3A identity and pairing

- Identity is created only after **Create identity**.
- The Ed25519 private seed is generated by Expo Crypto and stored only in Expo SecureStore.
- SQLite stores public identity, trust, revocation, and pairing replay metadata.
- Validation never creates trust. Riders compare the displayed code and explicitly confirm.
- Missing private keys are never regenerated silently.
- Reset clears identity/trust/pairing; complete local deletion also clears CrewLink data.
- Copy/paste and the local second-rider simulator are development flows, not a production transport or another physical phone.

Android runtime acceptance passed on `emulator-5554`. iOS runtime is unverified.

## Native Android development

`apps/mobile/android/` is generated Expo prebuild output and is not source-controlled. On Windows, the accepted native build uses the short `X:` project mapping and external Gradle caches to avoid CMake path-length/cache-lock problems. The installed development APK can use Metro on port 8081 for TypeScript-only updates.

Never commit generated native projects, APKs, database snapshots, logs, debug keystores, raw invitations, or private material.

## Architecture

- `apps/mobile`: screen composition and native adapters.
- `packages/crew-identity`: portable identity, Ed25519, canonical pairing, and platform-neutral lifecycle logic.
- `packages/crew-protocol`: versioned CrewLink wire schemas.
- `packages/crewlink`: canonical repository, migrations, simulators, retry, and coordination.
- `packages/core`, `packages/offline-packs`, `packages/ui`, and `packages/gpx`: existing domain/UI contracts.
- `apps/relay`: loopback-only development relay with no production authentication.

Unknown source/access states remain unknown. A route line never implies riding permission.
